Create Navigational Links within a Website

There are several techniques to navigate within a website. The semantic tag for navigation is the <nav> tag. You can navigate through your website by using relaltive addressing. For our examples we will use the unordered list tag (<ul>)

  • Page to page within the same folder
  • Page to page between folders
  • Staying on the current page
  • Going to a specific place on the current page (on page navigation)

Navigational menus are placed in your <header> semantic area. They may also be placed in your <footer> area as a site map. The nav tag, like other semantic tags, should have a <hx> tag, HOWEVER this header is optional. If you omit the header tag you will recieve a document outliner error: Untitled Section. Until you are shown how to avoid this error message, it will be acceptable on your document outline.

CSS Codiing Styles

Browser Defaults (no CSS) You have no control over the styling
Inline This is for a single specific element (tag) instance. Your CSS will be specified in a parameter within the element. This CSS affects ONLY the element tag that contains the specification. <p>Style="rule;[rules...]"</p> <p>style="colour:red; width:50px;"</p>
Embedded The specification of CSS as embedded happens in your <head> area. This CSS affects all similar elements on the web page where the the CSS is coded. <style> selector {rule; [rule; ...]}{selector{rule; [rule; ...]} ...</style> <style> p {colour:red, width:50px;} h1 {font-style:italic}</style>
Link (StyleSheet) This is the preferred method of doing CSS. You place you CSS in a seperate physical file. This file can then be used on ANY web page in your website. The link to this file will be done in your <head> area. The CSS in this file is similar to your embedded EXCEPT there is no <style> tag. selector {rule; [rule; ...]}{selector{rule; [rule; ...]} ... p {colour:red, width:50px;} h1 {font-style:italic}
